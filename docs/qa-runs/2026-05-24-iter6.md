## Bugs Filed This Iteration

None. All failures were test-script bugs (not application regressions), resolved
within this iteration via spec fixes.

### Test-script fixes applied

1. **Onboarding overlay CSS injection** (`profile.auth.spec.js`): The "Let's get
   started" overlay (z-50 fixed) appears on fresh accounts and intercepts pointer
   events. `dispatchEvent` and DOM removal were unreliable (React re-renders). Fix:
   inject `display:none !important; pointer-events:none` via `page.addStyleTag` after
   the page settles. CSS injection survives React re-renders.

2. **Roadmap overlay + force:true** (`roadmap.auth.spec.js`): Overlay check moved
   to AFTER `planTab.waitFor` (overlay appears after page load, not before). All
   sub-tab clicks use `{ force: true }` to bypass residual overlay interception.

3. **Roadmap strict-mode violation** (`roadmap.auth.spec.js`): `getByRole('button',
   { name: 'Scholarships' })` resolved to 2 elements when profile data was loaded
   (tab button + scholarship list item). Fixed: `exact: true` + `.first()` on all
   roadmap sub-tab locators.

4. **Discover overlay update** (`discover.auth.spec.js`): Updated from old
   `.fixed.inset-0` locator pattern to same `dispatchEvent` approach as other specs.
