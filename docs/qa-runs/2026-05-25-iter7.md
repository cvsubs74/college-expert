## Test-script fixes applied this iteration

### Fix 1: `profile_upload_pdf_processes_to_completion` — false positive assertion

**File:** `tests/playwright-prod/specs/profile.auth.spec.js`

**Root cause:** The success assertion chain included `or(page.getByText(/complete/i))` as
a fallback. The Profile page has a UI element "Complete transcript with courses, grades,
and course levels" (a tab label). This text was already visible BEFORE any upload occurred,
causing the locator to match in ~2s — a false positive. The test appeared to pass but no
actual upload success had occurred.

**Diagnosis:** Observed via Playwright trace inspection (action log showing the full upload
flow ran AND the assertion matched almost immediately after the upload button click) plus
API verification (`GET /get-profile` returning 404 after the "passing" test confirmed no
profile was created during the false-positive run).

**Fix:**
1. Removed the `/complete/i` fallback entirely. Only `successfully uploaded` and
   `upload.*success` patterns are now asserted.
2. Added `.first()` to suppress a secondary strict-mode violation: the `or()` locator
   was resolving to both a container `<div>` and an inner `<p>` element containing the
   success text.

**Verification:** After the fix, the test runs in 3.7–8.1s (consistent with actual upload +
Gemini processing), and profile exists at API level post-test.
